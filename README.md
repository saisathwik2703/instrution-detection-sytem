# Intrusion Detection System (Machine Learning)

A machine learning-powered system that detects and differentiates between
normal internet traffic and potential cyberattacks in real time, built
around the NSL-KDD benchmark dataset and served through an interactive
Streamlit web application.

## Features

- **Two classification models** — Decision Tree and Random Forest — trained
  on the NSL-KDD dataset to recognize attack patterns.
- **Real-time predictions** through a Streamlit web app: enter network
  traffic feature values (or pull a real sample from the test set) and get
  an instant safe / suspicious verdict with confidence score.
- **Binary classification**: normal traffic vs. attack traffic, covering
  the four NSL-KDD attack families — DoS, Probe, R2L, and U2R — collapsed
  into a single "attack" class.
- **Data pipeline** built with Pandas and NumPy: categorical encoding for
  protocol/service/flag fields, feature scaling, and a fitted-encoder
  save/load step so live input goes through the exact same transformation
  as training data.
- **Model persistence** with Joblib so trained models load instantly
  instead of retraining on every app launch.

## About the Dataset

[NSL-KDD](https://www.unb.ca/cic/datasets/nsl.html) is the cleaned-up,
widely-used successor to the original KDD Cup 1999 intrusion detection
dataset. Each record describes one network connection using 41 features
(connection duration, protocol, bytes transferred, login behavior, and
2-second/100-connection traffic-rate statistics) and is labeled either
`normal` or with one of ~22 specific attack types belonging to four
categories:

| Category | Meaning | Examples |
|---|---|---|
| DoS | Denial of Service | neptune, smurf, back, teardrop |
| Probe | Surveillance/scanning | satan, ipsweep, portsweep, nmap |
| R2L | Remote-to-local unauthorized access | guess_passwd, ftp_write, warezclient |
| U2R | User-to-root privilege escalation | buffer_overflow, rootkit, loadmodule |

This project collapses all attack types into a single binary target
(`0 = normal`, `1 = attack`) since the brief calls for distinguishing
*"normal internet traffic and potential cyberattacks"* — i.e. a safe vs.
suspicious classifier, not a 23-way classifier.

The `data/` folder already contains the real NSL-KDD train and test files
(pulled from the dataset's public NSL-KDD mirror) so the project runs
end-to-end with no extra downloads.

## How to Run

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Train the models
```bash
cd src
python train_models.py
```
This preprocesses the data, trains the Decision Tree and Random Forest,
prints accuracy/precision/recall/F1 plus a confusion matrix for each, and
saves everything needed by the app into `models/`.

### 3. Launch the web app
```bash
streamlit run app.py
```
Then open the local URL Streamlit prints (typically `http://localhost:8501`).

## Using the App

The app has three tabs:

1. **Check traffic** — a form covering all 41 NSL-KDD features (connection
   basics, content/behavior flags, and traffic-rate statistics), grouped
   into logical sections so it doesn't feel like a 41-field wall. Submitting
   it runs both models and shows each verdict side by side with a
   confidence percentage.
2. **Try a real sample** — pulls an actual labeled connection record from
   the held-out NSL-KDD test set, shows the true label, and runs both
   models against it — useful for demonstrating the system live without
   having to fill in 41 fields by hand.
3. **Model performance** — displays the saved accuracy/precision/recall/F1
   summary from training.

## Model Performance — two numbers worth knowing

This project reports performance two different ways, and the gap between
them is a well-known, deliberate property of NSL-KDD — worth understanding
rather than a bug:

| Evaluation method | Decision Tree | Random Forest |
|---|---|---|
| **Official NSL-KDD test split** (KDDTest+, attack types not seen in training) | ~77% accuracy | ~77% accuracy |
| **Random 80/20 split of the training set** (same distribution as training) | ~99.8% accuracy | ~99.9% accuracy |

`train_models.py` evaluates on the **official test split** by default,
because that's the standard, harder benchmark for this dataset and gives
an honest measure of how the model handles attack patterns it has never
seen — exactly the real-world scenario an intrusion detection system needs
to handle. The ~99% figure widely quoted for NSL-KDD projects comes from
evaluating on a random split of the training file instead, which lets the
model see similar attack variants during training, inflating the score.

Both numbers are legitimate to cite — just be clear about which split
produced which number (this matters if you're asked about it in a review
or interview).

## Possible Extensions

- Add a multi-class mode showing the predicted attack *category* (DoS,
  Probe, R2L, U2R) rather than just normal/attack.
- Add an XGBoost or neural-network model for comparison.
- Add SHAP-based explainability so the app can show *why* a connection
  was flagged, not just the verdict.
- Stream live `tcpdump`/Wireshark captures into the same feature pipeline
  for genuine real-time monitoring instead of manual/sample input.
